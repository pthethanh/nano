# Agent Workflow

Use this order by default:
1. Read [`knowledge/index.md`](../index.md).
2. Read the most relevant pages under `knowledge/wiki/`.
3. Read nearby package files and tests.
4. Make the smallest change that solves the task.
5. Update the knowledge base for durable findings or instructions.
6. Append a short entry to [`knowledge/log.md`](../log.md).

Knowledge-base maintenance rules:
- update existing pages instead of creating duplicates when possible
- keep pages concise and cross-linked
- keep `raw/` notes immutable unless explicitly refreshing them
- treat `index.md` and `log.md` as required maintenance surfaces, not optional docs
